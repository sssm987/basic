# basic

주문/재고 흐름에서 동시성 문제가 어떻게 생기는지 확인해보려고 만든 간단한 토이 프로젝트입니다.

처음에는 락을 적용하지 않은 상태로 두고, 같은 상품에 여러 요청이 동시에 들어올 때 어떤 문제가 생기는지 재현하는 용도로 만들었습니다.

이후에는 낙관적 락, 비관적 락을 각각 적용해서 동작 차이를 비교해보는 식으로 확장해볼 생각입니다.

## 사용 기술

- Java 21
- Spring Boot
- Spring Data JPA
- H2
- PostgreSQL
- Swagger

## 실행

로컬에서는 H2로 실행됩니다.

```bash
./gradlew bootRun
```

## 프로필

- local: H2
- prod: PostgreSQL

## Swagger

- http://localhost:8080/swagger-ui.html

## 현재 구현 범위

- 상품 더미 데이터 자동 적재
- 주문 생성 API
- 공통 에러 응답
- 재고 차감 로직
- 락 미적용 상태에서의 재고 문제 확인용 구조

## 프로젝트 목적

실서비스용 코드를 만드는 것보다는, 동시성 이슈를 설명하고 직접 확인해보기 위한 예제 프로젝트에 가깝습니다.
